## 1. 🔐 Dostęp do API

### Klucz API
```python
# .env
ANTHROPIC_API_KEY="sk-ant-..."

# Python
from dotenv import load_dotenv
import anthropic

load_dotenv()
client = anthropic.Anthropic()  # automatycznie czyta z env
```

> [!CAUTION]
> **NIGDY** nie umieszczaj klucza API w kodzie klienckim (frontend). Wszystkie requesty przez serwer!

### Podstawowy request
```python
response = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=1024,
    messages=[
        {"role": "user", "content": "Cześć, Claude!"}
    ]
)
print(response.content[0].text)
```

**Obowiązkowe parametry:** `model`, `max_tokens`, `messages`

### Jak Claude przetwarza request
1. **Tokenizacja** — tekst → tokeny
2. **Embedding** — tokeny → wektory liczbowe
3. **Kontekstualizacja** — zrozumienie kontekstu
4. **Generowanie** — przewidywanie kolejnych tokenów

### Kiedy Claude kończy generować?
- Osiągnięto `max_tokens`
- Naturalny koniec odpowiedzi (`end_turn`)
- Napotkano `stop_sequence`

## 2. 💬 Konwersacje wieloturowe

> [!IMPORTANT]
> Claude jest **bezstanowy** — nie pamięta poprzednich wiadomości! Musisz ręcznie zarządzać historią.

### Funkcje pomocnicze
```python
messages = []

def add_user_message(messages, content):
    messages.append({"role": "user", "content": content})

def add_assistant_message(messages, content):
    messages.append({"role": "assistant", "content": content})

def chat(messages, system=None, temperature=1.0, 
         stop_sequences=[], tools=None):
    params = {
        "model": "claude-sonnet-4-20250514",
        "max_tokens": 8096,
        "messages": messages,
    }
    if system:
        params["system"] = system
    if temperature != 1.0:
        params["temperature"] = temperature
    if stop_sequences:
        params["stop_sequences"] = stop_sequences
    if tools:
        params["tools"] = tools
    
    response = client.messages.create(**params)
    
    # Zachowaj pełną odpowiedź w historii
    add_assistant_message(messages, response.content)
    return response

# Ekstrakcja tekstu z odpowiedzi
def text_from_message(message):
    return "".join(
        block.text for block in message.content 
        if hasattr(block, "text")
    )
```

### Użycie
```python
messages = []
add_user_message(messages, "Czym jest Python?")
response = chat(messages)
print(text_from_message(response))

# Kontynuacja rozmowy
add_user_message(messages, "A jakie ma zastosowania?")
response = chat(messages)
```

## 5. 🌊 Streaming odpowiedzi

```python
# Metoda 1: stream=True
response = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=1024,
    messages=messages,
    stream=True
)
for event in response:
    if hasattr(event, 'delta') and hasattr(event.delta, 'text'):
        print(event.delta.text, end="", flush=True)

# Metoda 2: context manager (rekomendowana)
with client.messages.stream(
    model="claude-sonnet-4-20250514",
    max_tokens=1024,
    messages=messages
) as stream:
    for text in stream.text_stream:
        print(text, end="", flush=True)
    
    final_message = stream.get_final_message()
```

## 6. 📦 Dane strukturalne (JSON, kod)

**Problem:** Claude dodaje dodatkowy tekst wokół JSON/kodu.

**Rozwiązanie:** Prefilling + stop_sequences

```python
# Wymuś czysty JSON
add_user_message(messages, "Podaj 3 stolice w formacie JSON")
add_assistant_message(messages, "```json")  # prefilling!

response = chat(messages, stop_sequences=["```"])
# Wynik: czysty JSON bez owijki
```

> [!TIP]
> **Prefilling** = dodanie początku odpowiedzi asystenta (np. ` ```json`). 
> **Stop sequence** = znak, po którym Claude przestaje generować (np. ` ``` `).
> Razem dają idealnie czysty output!

## 7. 📊 Ewaluacja promptów

### Cykl optymalizacji
```
Prompt → Zbiór testowy → Claude → Ocena wyników → Zmiana promptu → Powtórz
```

### Pipeline ewaluacji
```python
def run_prompt(user_message, system_prompt):
    messages = []
    add_user_message(messages, user_message)
    return chat(messages, system=system_prompt)

def run_test_case(test_case, system_prompt, graders):
    response = run_prompt(test_case["input"], system_prompt)
    output = text_from_message(response)
    results = [grader(output, test_case) for grader in graders]
    return {"output": output, "results": results}

def run_eval(test_cases, system_prompt, graders):
    return [run_test_case(tc, system_prompt, graders) 
            for tc in test_cases]
```

### Rodzaje graderów

| Typ | Zastosowanie | Przykład |
|-----|-------------|---------|
| **Code grader** | Walidacja formatu | `json.loads(output)`, regex, `ast.parse()` |
| **Model grader** | Ocena jakości | Inny model AI ocenia output (np. Haiku) |
| **Human grader** | Finalna weryfikacja | Ręczny przegląd |

## 9. 🔧 Tool Use (Użycie narzędzi)

### Jak to działa?
```
User → Claude (decyduje o narzędziu) → Twój serwer (wykonuje) → Claude (odpowiada)
```

### Definicja narzędzia (schema)
```python
tools = [{
    "name": "get_current_datetime",
    "description": "Returns the current date and time in ISO format.",
    "input_schema": {
        "type": "object",
        "properties": {
            "timezone": {
                "type": "string",
                "description": "IANA timezone, e.g. 'Europe/Warsaw'"
            }
        },
        "required": ["timezone"]
    }
}]
```

### Funkcja narzędzia
```python
from datetime import datetime
from zoneinfo import ZoneInfo

def get_current_datetime(timezone: str) -> str:
    tz = ZoneInfo(timezone)
    return datetime.now(tz).isoformat()
```

### Obsługa tool use w konwersacji
```python
# Claude odpowiada z tool_use block
for block in response.content:
    if block.type == "tool_use":
        tool_name = block.name
        tool_input = block.input
        tool_id = block.id
        
        # Wykonaj narzędzie
        result = get_current_datetime(**tool_input)
        
        # Wyślij wynik z powrotem
        add_user_message(messages, [{
            "type": "tool_result",
            "tool_use_id": tool_id,
            "content": result
        }])
        
        # Claude teraz sformułuje finalną odpowiedź
        response = chat(messages, tools=tools)
```

### Pętla multi-turn z narzędziami
```python
while response.stop_reason == "tool_use":
    tool_results = []
    for block in response.content:
        if block.type == "tool_use":
            result = execute_tool(block.name, block.input)
            tool_results.append({
                "type": "tool_result",
                "tool_use_id": block.id,
                "content": result
            })
    
    add_user_message(messages, tool_results)
    response = chat(messages, tools=tools)
```

> [!IMPORTANT]
> Zachowuj **całą** odpowiedź Claude'a (wszystkie bloki!) w historii — nie tylko tekst!

## 10. 🔍 RAG (Retrieval Augmented Generation)

### Pipeline RAG w 5 krokach
1. **Chunk** — podziel dokument na fragmenty
2. **Embed** — zamień fragmenty na wektory (embeddingi)
3. **Store** — zapisz w bazie wektorowej
4. **Query** — zamień pytanie użytkownika na embedding
5. **Search** — znajdź najbardziej podobne fragmenty

### Strategie chunkingu

| Strategia | Kiedy użyć | Kod |
|-----------|-----------|-----|
| **Rozmiar** | Fallback, działa ze wszystkim | `text[i:i+chunk_size]` |
| **Zdania** | Dobry kompromis | `re.split(r'(?<=[.!?])\s+', text)` |
| **Struktura** | Markdown, dobrze sformatowane docs | `re.split(r'\n## ', text)` |
| **Semantyczny** | Najlepsza jakość, najdroższy | NLP grouping |

### Embeddingi (VoyageAI)
```python
import voyageai
client_v = voyageai.Client()

def generate_embedding(text, model="voyage-3-large"):
    result = client_v.embed([text], model=model, input_type="query")
    return result.embeddings[0]
```

### Hybrid Search = Semantic + BM25
```python
# Semantic: rozumie znaczenie, może pominąć dokładne terminy
# BM25: znajduje dokładne dopasowania, nie rozumie kontekstu
# Razem: Reciprocal Rank Fusion (RRF)

retriever = Retriever(VectorIndex(), BM25Index())
for chunk in chunks:
    retriever.add_document({"content": chunk})
results = retriever.search("zapytanie", k=3)
```

> **Cosine similarity:** 1 = identyczne, 0 = brak związku, -1 = przeciwne

## 14. 💾 Prompt Caching

### Kiedy używać?
- Wielokrotne wysyłanie **tego samego** system promptu / tool schemas
- Cache żyje **1 godzinę**
- Min. **1024 tokenów** do cache'owania

### Implementacja
```python
# Cache na system prompcie
params["system"] = [{
    "type": "text",
    "text": system_prompt,
    "cache_control": {"type": "ephemeral"}  # ← breakpoint!
}]

# Cache na narzędziach (ostatnie narzędzie)
tools_clone = tools.copy()
last_tool = tools_clone[-1].copy()
last_tool["cache_control"] = {"type": "ephemeral"}
tools_clone[-1] = last_tool
params["tools"] = tools_clone
```

**Kolejność cache:** Narzędzia → System prompt → Wiadomości
**Max breakpointy:** 4

> [!TIP]
> Nawet **jeden zmieniony znak** inwaliduje cache! Idealne dla stałych system promptów i schematów narzędzi.

## 16. 🔌 Model Context Protocol (MCP)

### Czym jest MCP?
MCP przenosi ciężar **definiowania i wykonywania narzędzi** z Twojego serwera na dedykowane serwery MCP.

### Serwer MCP (Python SDK)
```python
from mcp.server.fastmcp import FastMCP
from pydantic import Field

mcp = FastMCP("DocumentMCP")

@mcp.tool(
    name="read_doc",
    description="Odczytaj dokument po ID."
)
def read_document(
    doc_id: str = Field(description="ID dokumentu")
):
    return docs[doc_id]

@mcp.resource("docs://documents", mime_type="application/json")
def list_docs() -> list[str]:
    return list(docs.keys())

@mcp.prompt(name="format", description="Formatuj dokument do MD.")
def format_document(doc_id: str = Field(description="ID dokumentu")):
    return [base.UserMessage(f"Przeformatuj dokument {doc_id} na markdown...")]
```

### Klient MCP
```python
async def list_tools(self):
    result = await self.session().list_tools()
    return result.tools

async def call_tool(self, tool_name, tool_input):
    return await self.session().call_tool(tool_name, tool_input)

async def read_resource(self, uri):
    result = await self.session().read_resource(AnyUrl(uri))
    return result.contents[0]
```

**3 komponenty MCP:** Tools (akcje), Resources (dane), Prompts (szablony)
**Testowanie:** `mcp dev mcp_server.py` → Inspector w przeglądarce

## 17. 🤖 Agenci vs Workflow

### Kiedy co?

| | **Workflow** | **Agent** |
|--|-------------|----------|
| **Kiedy** | Znasz dokładne kroki | Nie wiesz jakie kroki będą potrzebne |
| **Zalety** | Przewidywalny, łatwy do testowania | Elastyczny, obsługuje nowe sytuacje |
| **Wady** | Mało elastyczny | Mniej niezawodny, trudniejszy do testowania |
| **Przykład** | Pipeline konwersji obrazu na model 3D | Asystent kodowania (Claude Code) |

### Wzorce workflow

#### Równoległość (Parallelization)
```
Obraz → [Ocena: metal] → Wynik 1 ─┐
      → [Ocena: polimer] → Wynik 2 ─┼→ Agregacja → Rekomendacja  
      → [Ocena: ceramika] → Wynik 3 ─┘
```
**Kiedy:** Niezależne oceny tego samego inputu

#### Łańcuchowanie (Chaining)
```
Krok 1: Napisz artykuł → Krok 2: Popraw błędy → Krok 3: Sformatuj
```
**Kiedy:** Claude ignoruje część instrukcji w długim prompcie. Rozdziel na fokusowane kroki!

#### Routing
```
Input → [Klasyfikacja: edukacja/rozrywka/comedy] → Dedykowany pipeline
```
**Kiedy:** Różne typy requestów wymagają zupełnie innego podejścia

#### Evaluator-Optimizer
```
Producer → Output → Grader → Feedback → Producer (powtarzaj aż OK)
```

### Inspekcja środowiska (kluczowe dla agentów!)
- Agent musi **widzieć wyniki swoich akcji** (screenshot po kliknięciu, plik po edycji)
- **Zawsze czytaj przed edycją** — Claude musi znać obecny stan
- Dawaj abstrakcyjne, kombinowalne narzędzia (bash, read, write, grep)

> [!TIP]
> **Najlepsza praktyka dla agentów:** Dostarczaj **abstrakcyjne narzędzia** zamiast ultra-specjalistycznych. Claude sam wymyśli jak je połączyć!
